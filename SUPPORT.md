# Getting help

## Documentation

Most questions are answered by the docs:

- [docs/](docs/README.md) — full documentation index
- [docs/getting-started.md](docs/getting-started.md) — install and first publish
- [docs/troubleshooting.md](docs/troubleshooting.md) — common errors and fixes

The top-level [README.md](README.md) is a quick overview; the docs go deeper.

## Bug reports and feature requests

Use the issue templates:

- [Bug report](.github/ISSUE_TEMPLATE/bug_report.md)
- [Feature request](.github/ISSUE_TEMPLATE/feature_request.md)

## Security issues

Don't open a public issue for security problems. See [SECURITY.md](SECURITY.md) for the private reporting flow.

## Discussion

For open-ended questions or design discussion, open a GitHub Discussion (if enabled) or a low-priority issue tagged "discussion".

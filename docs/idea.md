# The idea

We wanted a place to organize knowledge the way an encyclopedia does — entries that link to each other, group under domains and subjects, and grow over time without becoming a tangle of notes.

Off-the-shelf wikis felt either too heavy (databases, accounts, hosting) or too lightweight (a flat folder of markdown). So we built our own: drafts in a directory hierarchy that becomes the taxonomy, a publish step that derives tags and links between pages, an index that's regenerated on every publish, and a static deploy that anyone can fork and run.

The result is a personal encyclopedia where the source is just markdown and the live site is a single static deploy — small enough to understand end-to-end, capable enough to host real content.
